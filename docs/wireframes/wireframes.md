# 실시간 Car Market 와이어프레임

완성된 실시간 Car Market 서비스와 요구사항 정의서를 기준으로 작성한 제출용 와이어프레임 문서이다. 실제 완성 UI를 고해상도로 복제하지 않고, 화면 구조와 사용자 흐름을 저충실도 텍스트 와이어프레임으로 설명한다.

## 1. 와이어프레임 개요

| 항목 | 내용 |
| --- | --- |
| 문서 목적 | 실제 구현 화면의 구조와 사용자 흐름 설명 |
| 기준 화면 | PC 화면 |
| 표현 방식 | 흰색, 회색, 검정 중심의 저충실도 와이어프레임 |
| 포함 화면 | 로그인, 회원가입, 차량 목록, 구매자/딜러 차량 상세, 차량 등록, 차량 수정, 상담방 목록, 실시간 상담 |
| 제외 화면 | 관리자 화면. 관리자 기능은 미구현 |

## 2. 사용자 흐름

### 구매자 흐름

```text
로그인 → 차량 검색 → 상세 조회 → 상담 요청 → 실시간 상담 → 상담방 나가기
```

- 구매자는 차량 목록에서 검색, 필터, 정렬을 사용해 차량을 찾는다.
- 차량 상세에서 이미지 갤러리와 주요 스펙을 확인한 뒤 상담을 요청한다.
- 상담방 나가기를 하면 본인 상담 목록에서만 숨김 처리된다.

### 딜러 흐름

```text
로그인 → 차량 등록 → 차량 관리 → 상담 응답 → 상담방 나가기
```

- 딜러는 차량 등록/수정/삭제를 수행한다.
- 본인 차량 상세에서는 상담하기 대신 수정/삭제 버튼을 확인한다.
- 구매자가 생성한 상담방에서 실시간 상담에 응답한다.

### 재상담 흐름

```text
상담방 나가기 → 내 상담 목록에서 숨김 → 같은 차량 상담하기 → 기존 상담방 재활성화
```

- 상담방 나가기는 상담방과 메시지를 삭제하지 않는다.
- 한쪽 사용자가 나가도 상대방 상담 목록과 기존 메시지는 유지된다.
- 같은 차량에서 다시 상담하면 기존 상담방이 다시 활성화된다.

## 4. 화면별 와이어프레임

### WF-01. 로그인

#### 화면 목적

사용자가 이메일과 비밀번호로 로그인하여 서비스 기능을 이용할 수 있게 한다.

#### 텍스트 기반 와이어프레임

```text
┌──────────────────────────────────────────┐
│ 실시간 Car Market        차량 목록        │
├──────────────────────────────────────────┤
│                                          │
│              [ 로그인 ]                  │
│                                          │
│              이메일 입력                 │
│              비밀번호 입력               │
│              [로그인 버튼]               │
│              회원가입 이동               │
│                                          │
└──────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 이메일 | Firebase Authentication 로그인 이메일 |
| 비밀번호 | Firebase Authentication 로그인 비밀번호 |
| 로그인 버튼 | 로그인 요청 실행 |
| 회원가입 이동 | 회원가입 화면으로 이동 |

#### 주요 동작

- 이메일과 비밀번호로 로그인한다.
- 로그인 성공 후 차량 목록 등 서비스 화면으로 이동한다.

#### 반응형 동작

- 모바일에서는 로그인 카드가 화면 폭에 맞게 줄어든다.

### WF-02. 회원가입

#### 화면 목적

사용자가 계정을 생성하고 구매자 또는 딜러 역할을 선택한다.

#### 텍스트 기반 와이어프레임

```text
┌──────────────────────────────────────────┐
│ 실시간 Car Market        차량 목록        │
├──────────────────────────────────────────┤
│                                          │
│             [ 회원가입 ]                 │
│                                          │
│             이름 입력                    │
│             이메일 입력                  │
│             비밀번호 입력                │
│             역할 선택: 구매자 / 딜러      │
│             [회원가입 버튼]              │
│                                          │
└──────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 이름 | 사용자 표시 이름 |
| 이메일 | 로그인 계정 이메일 |
| 비밀번호 | 로그인 비밀번호 |
| 역할 선택 | `buyer` 또는 `dealer` 역할 선택 |
| 회원가입 버튼 | Firebase 계정 생성 및 MongoDB 사용자 프로필 저장 |

#### 주요 동작

- 사용자는 구매자 또는 딜러 역할을 선택해 가입한다.
- 회원가입 성공 시 로그인 상태로 서비스 이용이 가능하다.

#### 반응형 동작

- 모바일에서는 입력 필드가 세로로 유지되고 화면 폭에 맞춰 정렬된다.

### WF-03. 차량 목록

#### 화면 목적

사용자가 차량을 검색, 필터, 정렬하고 페이지 단위로 탐색할 수 있게 한다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market     차량 목록 / 상담 목록     사용자 │
├───────────────┬────────────────────────────────────────┤
│ 차량 조건     │ 차량명 검색                  정렬 선택 │
│ n대 초기화    ├────────────────────────────────────────┤
│ 차종 필터     │ 차량 카드 3열                            │
│ 제조사 필터   │ ┌──────┐ ┌──────┐ ┌──────┐              │
│ 연식 필터     │ │카드  │ │카드  │ │카드  │              │
│ 가격 필터     │ │상담  │ │상담  │ │상담  │              │
│ 지역 필터     │ └──────┘ └──────┘ └──────┘              │
│ 연료 필터     │ ┌──────┐ ┌──────┐ ┌──────┐              │
│ 변속기 필터   │ │카드  │ │카드  │ │카드  │              │
│               │ └──────┘ └──────┘ └──────┘              │
│               │       이전 | 1 | 2 | 3 | 다음          │
└───────────────┴────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 헤더 | 로고, 차량 목록, 차량 등록, 상담 목록, 사용자명, 로그아웃 |
| 차량명 검색 | 차량명 기준 검색 |
| 정렬 | 최신순, 가격순, 최신 연식순, 낮은 주행거리순 |
| 왼쪽 조건 필터 | 조건에 맞는 차량 수, 초기화, 제조사/차종/가격/연식/주행거리/지역/연료/변속기 필터 |
| 차량 카드 | 차량 이미지, 차량명, 주요 정보, 가격, 상담하기 버튼 |
| 페이지네이션 | 검색/필터 결과를 9개 단위로 표시 |

#### 주요 동작

- 필터와 검색 조건은 변경 즉시 결과에 반영된다.
- 정렬 변경 시 1페이지로 초기화된다.
- 와이어프레임에는 대표 카드 6개만 표현하지만 실제 서비스에서는 한 페이지당 최대 9개를 표시한다.
- 차량 카드 전체 클릭 시 상세페이지로 이동한다.
- 상담하기 버튼은 상담방 생성 또는 기존 상담방 이동을 실행한다.

#### 반응형 동작

- PC: 필터 사이드바와 차량 카드 3열.
- 태블릿: 차량 카드 2열.
- 모바일: 차량 카드 1열, 필터와 검색 영역은 세로 배치.

### WF-04. 구매자 차량 상세

#### 화면 목적

구매자가 차량 상세 정보와 이미지를 확인하고 딜러에게 상담을 요청한다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market       차량 목록 / 상담 목록   구매자 │
├──────────────────────────────┬─────────────────────────┤
│ 메인 이미지                  │ 차량 요약 카드          │
│  ←                      →    │ 차량명 / 판매가         │
│                              │ 연식 / 주행거리 / 지역  │
│ 썸네일 썸네일 썸네일         │ 담당 딜러               │
│                              │ [상담하기]              │
│ 주요 스펙 4열                │ [목록으로 돌아가기]     │
│ 상세 설명                    │                         │
└──────────────────────────────┴─────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 메인 이미지 | 차량 대표 이미지 표시 |
| 이전/다음 화살표 | 이미지 순환 이동 |
| 썸네일 | 선택 시 메인 이미지 변경 |
| 차량 요약 카드 | 차량명, 가격, 연식, 주행거리, 지역 |
| 담당 딜러 | 차량 담당 딜러명 표시 |
| 상담하기 | 상담방 생성 또는 기존 상담방 이동 |
| 주요 스펙 | 제조사, 연식, 차종, 연료, 변속기, 주행거리, 지역, 색상 |
| 상세 설명 | 차량 설명 문단 표시 |

#### 주요 동작

- 썸네일, 화살표, 키보드 방향키, 모바일 스와이프로 이미지가 이동한다.
- 구매자는 상담하기 버튼으로 딜러와 상담을 시작한다.

#### 반응형 동작

- PC: 좌측 이미지/스펙/설명, 우측 요약 카드.
- 모바일: 이미지 → 요약 카드 → 주요 스펙 → 상세 설명 순서로 세로 배치.

### WF-05. 딜러 차량 상세

#### 화면 목적

딜러가 본인 차량의 상세 정보를 확인하고 수정 또는 삭제를 수행한다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market  차량 목록 / 차량 등록 / 상담 목록  │
├──────────────────────────────┬─────────────────────────┤
│ 메인 이미지                  │ 차량 요약 카드          │
│  ←                      →    │ 차량명 / 판매가         │
│                              │ 등록 딜러               │
│ 썸네일 썸네일 썸네일         │ [수정] [삭제]           │
│                              │ [목록으로 돌아가기]     │
│ 주요 스펙 4열                │                         │
│ 상세 설명                    │                         │
└──────────────────────────────┴─────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 이미지 갤러리 | 구매자 상세와 동일한 갤러리 구조 |
| 등록 딜러 | 본인 차량 화면에서 딜러 라벨 표시 |
| 수정/삭제 | 본인이 등록한 차량에만 표시 |
| 주요 스펙/상세 설명 | 구매자 상세와 동일한 정보 구조 |

#### 주요 동작

- 딜러 본인 차량에서는 상담하기 버튼이 표시되지 않는다.
- 수정 버튼은 차량 수정 화면으로 이동한다.
- 삭제 버튼은 차량 삭제를 실행한다.

#### 반응형 동작

- PC: 좌우 2열 상세 구조.
- 모바일: 이미지 → 요약 카드 → 주요 스펙 → 상세 설명 순서.

### WF-06. 차량 등록

#### 화면 목적

딜러가 차량 기본 정보, 상세 설명, 이미지를 입력하여 차량을 등록한다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market  차량 목록 / 차량 등록 / 상담 목록  │
├────────────────────────────────────────────────────────┤
│ 차량 정보 입력 폼                                     │
│ 차량명 | 제조사 | 가격 | 연식                         │
│ 차종   | 연료   | 주행거리 | 지역                     │
│ 변속기 | 외장 색상                                    │
│ 설명                                                   │
│ 이미지 최소 1장 · 최대 8장                            │
│ [직접 업로드] [샘플 이미지 선택 활성]                 │
│ 샘플 차량 목록 3열                                    │
│ 선택 이미지 목록                                      │
│ [취소]                                      [차량 등록]│
└────────────────────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 접근 권한 | 딜러 계정만 접근 가능 |
| 차량 정보 입력 폼 | 차량명, 제조사, 가격, 연식, 차종, 연료, 주행거리, 지역, 변속기, 외장 색상 |
| 설명 | 차량 상태와 특이사항 입력 |
| 직접 업로드 | 파일 선택 input으로 이미지 업로드. 실제 기본 탭은 직접 업로드 |
| 샘플 이미지 선택 | 샘플 차량 목록에서 이미지를 클릭 순서대로 선택. 와이어프레임은 샘플 목록 표시 상태 |
| 선택 이미지 목록 | 선택한 이미지 순번과 파일명 표시 |
| 이미지 필수 조건 | 직접 업로드 이미지와 샘플 이미지를 합산하여 최소 1장 이상, 최대 8장까지 선택 |

#### 주요 동작

- 이미지가 0장이면 등록 요청을 보내지 않고 오류 메시지를 표시한다.
- 샘플 이미지는 클릭 순서대로 `imageUrls`에 저장된다.
- 첫 번째 이미지가 대표 이미지 `imageUrl`이 된다.

#### 반응형 동작

- PC: 입력 폼 2열, 샘플 차량 목록 3열.
- 모바일: 입력 필드와 샘플 카드가 1열 또는 2열로 재배치된다.

### WF-07. 차량 수정

#### 화면 목적

딜러가 본인이 등록한 차량의 정보와 이미지를 수정한다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market  차량 목록 / 차량 등록 / 상담 목록  │
├────────────────────────────────────────────────────────┤
│ 기존 차량 정보                                        │
│ 차량명 | 제조사 | 가격 | 연식                         │
│ 차종   | 연료   | 주행거리 | 지역                     │
│ 변속기 | 외장 색상                                    │
│ 설명                                                   │
│ 이미지 관리                                           │
│ 최종 이미지 최소 1장 · 최대 8장                       │
│ 기존 이미지 [삭제]  새 업로드 [삭제]  샘플 [삭제]     │
│ [직접 업로드] [샘플 이미지 선택]                      │
│ 샘플 차량 목록                                        │
│ [취소]                                      [수정 완료]│
└────────────────────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 접근 권한 | 딜러 계정만 접근 가능하며 본인이 등록한 차량만 수정 가능 |
| 기존 차량 정보 | 기존 저장값을 입력 필드에 표시 |
| 기존 이미지 | 삭제하지 않은 기존 이미지는 유지 |
| 새 이미지 | 직접 업로드 또는 샘플 이미지로 추가 가능 |
| 이미지 필수 조건 | 삭제하지 않은 기존 이미지와 새 업로드·샘플 이미지를 합산하여 최소 1장 이상 유지 |
| 수정 완료 | 기존 유지 이미지 뒤에 새 이미지가 추가된 순서로 저장 |

#### 주요 동작

- 기존 이미지를 모두 삭제하고 새 이미지를 추가하지 않으면 저장 요청을 차단한다.
- 최종 이미지 수는 최소 1장, 최대 8장이다.
- 대표 이미지는 최종 이미지 목록의 첫 번째 이미지로 유지된다.

#### 반응형 동작

- PC: 입력 폼 2열, 이미지 목록은 카드 그리드.
- 모바일: 입력 필드와 이미지 카드가 세로 배치된다.

### WF-08. 상담방 목록

#### 화면 목적

구매자와 딜러가 본인의 상담방을 확인하고 입장하거나 상담방을 나간다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market       차량 목록 / 상담 목록   사용자 │
├────────────────────────────────────────────────────────┤
│ 상담 목록 | 총 n개의 상담방 | 차량 목록 이동           │
├────────────────────────────────────────────────────────┤
│ 차량명 + 상대방 이름     최근 메시지     시간 [나가기] │
│ 차량명 + 상대방 이름     최근 메시지     시간 [나가기] │
└────────────────────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 상담방 수 | 현재 사용자에게 표시되는 상담방 수 |
| 상담방 카드 | 차량명, 상대방 이름, 최근 메시지, 최근 시간 |
| 상담방 나가기 | 확인 후 해당 사용자의 목록에서만 숨김 |
| 차량 목록 이동 | 차량 목록 화면으로 이동 |

#### 주요 동작

- 상담방 카드를 클릭하면 실시간 상담 화면으로 이동한다.
- 나가기 버튼 클릭 시 확인창을 표시한다.
- 나가도 상대방 상담방과 기존 메시지는 유지된다.
- 같은 차량에서 다시 상담하면 기존 상담방이 재활성화된다.

#### 반응형 동작

- PC: 상담방 정보와 나가기 버튼이 한 줄에 배치된다.
- 모바일: 상담방 정보, 시간, 나가기 버튼이 세로 또는 줄바꿈 배치된다.

### WF-09. 실시간 상담

#### 화면 목적

구매자와 딜러가 같은 상담방에서 실시간 메시지를 주고받는다.

#### 텍스트 기반 와이어프레임

```text
┌────────────────────────────────────────────────────────┐
│ 실시간 Car Market       차량 목록 / 상담 목록   사용자 │
├────────────────────────────────────────────────────────┤
│ 차량명                                                  │
│ 딜러/구매자 표시 + 딜러 온라인 상태        [나가기]    │
├────────────────────────────────────────────────────────┤
│ 상대방 메시지                              시간         │
│                                      내 메시지  시간    │
│ 상대방 메시지                              시간         │
├────────────────────────────────────────────────────────┤
│ 메시지 입력창                         [전송 버튼]      │
│ Enter 전송 / Shift+Enter 줄바꿈                       │
└────────────────────────────────────────────────────────┘
```

#### 주요 구성 요소

| 요소 | 설명 |
| --- | --- |
| 차량명 | 상담 중인 차량명 |
| 상대방 표시 | 구매자는 딜러명과 온라인 상태, 딜러는 구매자명 표시 |
| 좌우 메시지 | 내 메시지는 오른쪽, 상대방 메시지는 왼쪽 |
| 입력창 | 메시지 입력 |
| 전송 버튼 | 메시지 전송 |
| 상담방 나가기 | 확인 후 해당 사용자 목록에서만 숨김 |

#### 주요 동작

- Socket.io로 메시지를 실시간 송수신한다.
- 메시지는 MongoDB에 저장된다.
- Enter는 전송, Shift+Enter는 줄바꿈이다.
- 상담방 나가기 후 상대방 상담방과 기존 메시지는 유지된다.
- 같은 차량에서 다시 상담하면 기존 상담방이 재활성화된다.

#### 반응형 동작

- PC: 메시지 영역을 넓게 표시한다.
- 모바일: 상단 정보와 나가기 버튼이 줄바꿈되고 메시지 폭이 화면에 맞춰 조정된다.

## 5. 구매자와 딜러 차량 상세 화면 차이

| 구분 | 구매자 차량 상세 | 딜러 차량 상세 |
| --- | --- | --- |
| 목적 | 차량 정보 확인 및 상담 요청 | 본인 차량 관리 |
| 딜러 표시 | 담당 딜러 | 등록 딜러 |
| 주요 버튼 | 상담하기 | 수정 / 삭제 |
| 상담 상태 | 상담 버튼 제공 | 본인 차량 상담 버튼 숨김 |
| 이미지/스펙/설명 | 동일한 구조 | 동일한 구조 |

## 6. 반응형 동작

| 화면 | PC | 태블릿 | 모바일 |
| --- | --- | --- | --- |
| 차량 목록 | 필터 사이드바 + 차량 카드 3열 | 차량 카드 2열 | 차량 카드 1열, 필터와 검색 영역 세로 배치 |
| 차량 상세 | 좌측 이미지/스펙/설명 + 우측 요약 카드 | 주요 영역 폭 조정 | 이미지 → 요약 카드 → 주요 스펙 → 상세 설명 순서 |
| 차량 등록 | 입력 폼 2열, 샘플 차량 목록 3열 | 샘플 카드 2열 가능 | 입력 필드와 샘플 카드 세로 배치 |
| 차량 수정 | 입력 폼 2열, 이미지 카드 그리드 | 이미지 카드 폭 조정 | 기존 이미지와 새 이미지가 세로 배치 |
| 상담방 목록 | 상담방 정보와 나가기 버튼 한 줄 | 줄바꿈 가능 | 정보, 시간, 나가기 버튼 세로 배치 |
| 실시간 상담 | 넓은 메시지 영역 | 메시지 폭 조정 | 상단 정보 줄바꿈, 입력창과 버튼 화면 폭 대응 |

## 7. 제외 범위

- 관리자 화면은 구현되지 않았으므로 포함하지 않는다.
- 메시지 삭제 기능은 구현되지 않았으므로 포함하지 않는다.
- 상담방 나가기는 상담방 삭제가 아니라 사용자별 목록 숨김 기능이다.
