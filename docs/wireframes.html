실시간 Car Market 와이어프레임

완성된 서비스와 요구사항 정의서를 기준으로 작성한 제출용 화면 구조 문서

서비스명실시간 Car Market 문서 목적실제 구현 화면의 구조와 사용자 흐름 설명 작성 기준React/Vite 클라이언트, Express 서버, MongoDB Atlas, Firebase Authentication, Socket.io 작성 형식저충실도 와이어프레임, A4 PDF 변환용 HTML

1. 와이어프레임 개요

본 문서는 완성된 실시간 Car Market 서비스의 화면 구조와 주요 인터랙션을 설명한다. 실제 브랜드 색상이나 완성 UI를 재현하기보다, 입력창, 버튼, 카드, 이미지 영역, 목록 구조를 단순 도형과 한글 라벨로 표현한다.

작성 원칙

항목 내용
표현 수준 흰색, 회색, 검정 중심의 저충실도 와이어프레임
기준 화면 PC 화면을 기본으로 작성
반응형 모바일과 태블릿에서는 주요 영역이 세로 배치되도록 설명
제외 범위 관리자 화면은 미구현이므로 포함하지 않음

10. 구매자와 딜러 화면 차이

구분 구매자 딜러
차량 상세 담당 딜러와 상담하기 버튼 표시 본인 차량에서는 등록 딜러, 수정, 삭제 버튼 표시
차량 관리 등록/수정/삭제 불가 본인 차량 등록/수정/삭제 가능
상담방 목록 본인이 생성한 상담방 표시 본인 차량에 연결된 상담방 표시
실시간 상담 딜러 온라인/오프라인 상태 표시 구매자 이름과 메시지 표시

11. 반응형 동작

화면 PC 태블릿/모바일
차량 목록 필터 사이드바 + 차량 카드 3열 카드 2열 또는 1열로 축소, 필터는 화면 폭에 맞춰 세로 배치
차량 상세 좌측 이미지/스펙 + 우측 요약 카드 2열 이미지, 요약 카드, 주요 스펙, 상세 설명 순서로 세로 배치
등록/수정 입력 폼 2열, 샘플 차량 목록 3열 입력 필드와 샘플 카드가 1열 또는 2열로 재배치
상담 상담방과 메시지 영역을 넓게 표시 상단 정보와 버튼이 줄바꿈되고 메시지 폭을 화면에 맞춤
본 와이어프레임은 실제 구현 화면의 구조와 흐름을 설명하기 위한 문서이며, 관리자 화면은 구현되지 않았으므로 포함하지 않는다.
